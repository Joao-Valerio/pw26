Camadas criadas

O projeto foi estruturado em camadas bem definidas: `models.py` para dominio, `forms.py` para estados e entrada de dados, `services.py` para montagem dos dados de dashboard e `views.py` para orquestrar as paginas.

Equivalencia com o diagrama

Componentes como `Sidebar`, `TabelaGastos`, `ChartAreaInteractive`, `ChartLineGastosCliente` e `ContactForm` foram transformados em templates reutilizaveis dentro de `website/templates/website/components`.

Rotas da aplicacao

A navegacao do sistema agora concentra `dashboard`, `gastos`, `metas`, `relatorios`, `recursos`, `contato`, `login` e `cadastro`, espelhando a organizacao que o diagrama sugeria para o frontend.
